# 📄 나만의 계약서 자동 생성기 — 시작 가이드

> 매번 한글/워드 파일 열어서 **이름·날짜·금액만 바꿔** 계약서를 다시 만드는 일,
> 이제 안 해도 돼요. 이 폴더는 "틀"이고, **당신 계약서로 갈아끼우면 당신 전용 도구**가 됩니다.
> 코딩 몰라도 됩니다. VS Code에서 **Claude한테 말로 시키면** 돼요.

---

## 0. 큰 그림 (1분이면 이해돼요)

폴더 번호 순서가 곧 흐름이에요:

```
1.신청서넣기  📥 구글폼 응답(CSV)이나 메모를 넣는 곳   ← 평소 여기에 넣어요
2.계약서양식  📄 계약서 틀(양식)                      ← 내 계약서로 한 번 갈아끼움
3.내정보      🙋 나·거래처 정보                       ← 한 번만 채우면 계속 재사용
4.완성계약서  ✅ 완성된 계약서가 여기서 나옴 (PDF)      ← 결과물
```

**당신이 바꾸는 건 ②양식과 ③내정보 딱 두 개**예요. 나머지는 넣는 곳(①)과 나오는 곳(④).

---

## 1. 준비물

- **VS Code + Claude Code** (이미 쓰고 계시죠)
- **내 계약서 원본 1개** — 워드(`.docx`)나 PDF
  - 한글(`.hwp`) 파일이면, 한글에서 `다른 이름으로 저장 → PDF`로 먼저 내보내 주세요. (이유는 2번 끝에)

---

## 2. VS Code 확장 깔기 (한 번만 — 폴더 안 파일을 편하게 보기)

폴더 안의 CSV·워드·이 가이드(.md)를 VS Code에서 바로 열어보려면 확장 몇 개가 편해요.

> 👇 **확장 이름을 클릭하면** 마켓플레이스 설치 페이지로 바로 가요.

| 확장 이름 | 뭘 해주나요 | 검색용 ID |
|---|---|---|
| **[Rainbow CSV](https://marketplace.visualstudio.com/items?itemName=mechatroner.rainbow-csv)** | 신청내역 CSV를 색깔로 보기 좋게 | `mechatroner.rainbow-csv` |
| **[Office Viewer](https://marketplace.visualstudio.com/items?itemName=cweijan.vscode-office)** | 워드(.docx)·엑셀·PDF를 VS Code에서 바로 열기 | `cweijan.vscode-office` |
| **[HWP Viewer](https://marketplace.visualstudio.com/items?itemName=edwardkim.rhwp-vscode)** | 한글(.hwp/.hwpx)을 VS Code에서 바로 보기 | `edwardkim.rhwp-vscode` |
| **[Office Editor](https://marketplace.visualstudio.com/items?itemName=shubhamprajapati1202.office-editor)** | 워드(.docx)·엑셀(.xlsx)을 VS Code 안에서 편집 | `shubhamprajapati1202.office-editor` |
| **[Markdown Preview Enhanced](https://marketplace.visualstudio.com/items?itemName=shd101wyy.markdown-preview-enhanced)** | 이 가이드(.md)를 예쁘게 미리보기 | `shd101wyy.markdown-preview-enhanced` |
| **[Markdown Editor](https://marketplace.visualstudio.com/items?itemName=adamerose.markdown-wysiwyg)** | .md를 워드처럼 보면서 바로 편집(WYSIWYG) | `adamerose.markdown-wysiwyg` |

**설치 방법** (둘 중 편한 걸로):
- 🖱️ 왼쪽 세로줄에서 **네모 4개 아이콘**(확장, `⇧⌘X`) 클릭 → 위 이름 검색 → **Install**
- ⌨️ 또는 VS Code 터미널에 한 줄씩:
  ```bash
  code --install-extension mechatroner.rainbow-csv
  code --install-extension cweijan.vscode-office
  code --install-extension edwardkim.rhwp-vscode
  code --install-extension shubhamprajapati1202.office-editor
  code --install-extension shd101wyy.markdown-preview-enhanced
  code --install-extension adamerose.markdown-wysiwyg
  ```

> 💡 **마크다운(.md) 미리보기는 사실 VS Code에 기본 내장**돼 있어요. `.md` 파일 연 상태에서
> `⇧⌘V`(Shift+Command+V)를 누르면 바로 미리보기가 떠요. 위 확장은 더 예쁘게 보고 싶을 때만.
>
> 💡 **한글(.hwp)도** 이제 위 **HWP Viewer**로 VS Code 안에서 바로 열어볼 수 있어요. (보기 전용)
> 다만 계약서를 **양식으로 등록(4번)**할 땐 Claude가 내용을 읽어야 해서, `.hwp`는 한글에서
> `PDF로 내보내기`를 한 번 해 주는 게 가장 확실해요. 보는 건 HWP Viewer, 자동화에 넣는 건 PDF —
> 이렇게 기억하면 돼요.

## 3. 스킬 설치 (한 번만 — 자동화의 두뇌)

이 자동화는 Claude Code **"스킬" 하나**로 움직여요. 이 폴더 안 `_스킬/contract-vary`를
Claude가 찾는 위치로 복사하면 됩니다.

**제일 쉬운 방법** — Claude한테 이렇게 말하세요:
> "이 폴더 `_스킬`에 있는 contract-vary 스킬을 내 `~/.claude/skills`에 설치해줘"

**직접 하고 싶으면** — VS Code 터미널(이 폴더에서):
```bash
mkdir -p ~/.claude/skills && cp -R _스킬/contract-vary ~/.claude/skills/
```

설치 후 VS Code(또는 Claude Code)를 한 번 끄고 켠 뒤, 입력창에 `/`를 쳤을 때
`contract-vary`가 보이면 완료예요.

---

## 4. 내 계약서를 "양식"으로 등록하기 ⭐ (제일 중요 · 한 번만)

여기가 핵심이에요. 내 계약서를 한 번만 "양식"으로 바꿔두면, 그다음부턴 값만 바뀝니다.

1. 내 계약서 원본(`.docx`/`.pdf`)을 이 폴더 아무 데나 둬요.
2. Claude한테 말하세요:
   > "내 계약서 `○○.pdf`를 양식으로 등록해줘"
3. Claude가 계약서를 읽고 **어디가 매번 바뀌는 자리인지**(이름·날짜·금액·장소 등)
   하나씩 같이 짚어줄 거예요. 맞는지 **확인만** 해주면 돼요.
4. 끝나면 `2.계약서양식/`에 내 양식이 생깁니다. (약관 본문은 그대로, 바뀌는 자리만 빈칸 처리)

> 이 "어디가 바뀌는 자리인지 고르는 일"이 자동화의 진짜 핵심이에요.
> Claude가 제안하지만, **내 눈으로 한 번 확인**하는 게 중요해요.

## 5. 내 정보 채우기 (한 번만)

> "내 정보랑 거래처 정보 채워줘"

라고 하면 Claude가 묻는 대로 답만 하면 돼요(이름·계좌·거래처 회사 정보 등).
`3.내정보/`에 저장돼서 **다음부턴 다시 안 물어봐요.**

---

## 6. 계약서 한 장 뽑아보기

> "○○ 건 계약서 하나 뽑아줘"

→ 바뀌는 값(날짜·건수·금액 등)만 답하면, `4.완성계약서/`에 **HTML과 PDF**가 같이 생겨요.
PDF 열어서 확인하고 고객에게 보내면 끝. (전자서명은 유캔싸인 등에 그 PDF를 올리면 돼요.)

---

## 7. (선택) 구글폼으로 여러 건 한 번에

고객이 구글폼으로 정보를 넣게 했다면:

1. 구글폼 응답을 **CSV로 내려받아** `1.신청서넣기/`에 넣어요.
2. Claude한테: **"신청내역 CSV 읽어서 전부 뽑아줘"**
3. 한 줄이 계약서 한 장이라, 여러 줄이면 **한 번에 여러 장**이 나와요.

## 🔁 재계약은요?

> "○○ 재계약, 날짜만 바뀜"

이라고 하면 **지난 계약을 불러와** 바뀐 것만 고쳐서 새 버전(v2)으로 저장해요.
이전 버전도 그대로 남아 있어서 "언제 뭘 보냈는지" 이력이 쌓입니다.

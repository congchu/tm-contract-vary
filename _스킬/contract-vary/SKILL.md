# contract-vary — 문서 배리에이션 + 파일 버전 관리

마스터 양식(HTML) 하나에서 🟡건별 값만 바꿔 최종본을 뽑고, `4.완성계약서/`에 파일로 남겨
버전을 관리한다. 본문(약관)은 절대 손대지 않는다 — 바뀌는 건 변수뿐이다.

## 폴더 구조 (일반인용 한글 이름. 파일이 곧 DB)

작업 폴더 기준 (예: `~/Documents/contract-vary/`):

```
1.신청서넣기/   📥 input. 구글폼 응답 CSV·메모를 넣는 곳. 한 줄 = 계약서 한 장.
2.계약서양식/   마스터 양식. {{변수}} 박힌 HTML. 평소엔 손대지 않음.
3.내정보/
   내정보.json   🔵 내 고정값 (강사/제공자 본인) — 항상 같음
   거래처.json   🟢 거래처별 정보 — 업체 키로 골라 씀
4.완성계약서/   📤 output = 이력. 날짜_거래처_종류_v버전.html
```

`4.완성계약서/` 폴더 그 자체가 이력이다. `ls`가 곧 계약 목록.
git 폴더면 커밋 히스토리로 누가·언제·뭘 바꿨는지까지 남는다.

(구버전 영어 폴더명 templates/ · _data/ · records/ 를 만나도 같은 역할로 취급한다.)

## 변수 3종 + 자동계산

- 🔵 **내 고정값** → `3.내정보/내정보.json`에서 그대로 채움 (다시 묻지 않는다)
- 🟢 **거래처** → `3.내정보/거래처.json`에서 업체 골라 채움. 없으면 묻고 **추가 제안**
- 🟡 **건별** → 매번 받는다 (교육과목·일정·건수·건당금액·계약일 등)
- ⚙️ **자동계산** → 총금액 = 건수 × 건당금액. 한글금액도 자동 변환

## 0. 양식 등록 (처음 한 번 — word/pdf 원본을 양식으로)

새 사용자가 자기 계약서(보통 `.docx`/`.pdf`)를 가져오면, 먼저 그걸 `{{변수}}` 박힌 HTML 양식으로
바꿔 `2.계약서양식/`에 등록한다. **처음 한 번만** 한다. 이후엔 이 양식을 재사용.

1. 원본 텍스트를 추출한다.
   - **PDF**: 그냥 읽는다(Read 툴이 PDF를 직접 읽음).
   - **Word(.docx)**: `textutil -convert txt "원본.docx" -output -` (macOS 기본 내장).
     없으면 `pandoc "원본.docx" -t plain`. 둘 다 없으면 사용자에게 PDF로 내보내 달라고 요청.
2. 사용자와 함께 **어디가 매번 바뀌는지** 짚는다(🟡건별 / 🟢거래처 / 🔵본인 / ⚙️자동계산).
   이 분리를 사용자가 직접 확인하게 한다 — 자동화의 본체다.
3. 약관 본문은 그대로 두고, 바뀌는 자리만 `{{변수}}`로 치환한 HTML을 만들어
   `2.계약서양식/<이름>.html`로 저장한다. A4 인쇄 CSS 포함(`@page`, 한글 폰트), 서명란·총금액 자리도 변수로.
4. 그 양식의 🔵는 `3.내정보/내정보.json`, 🟢는 `거래처.json`에 한 번 채워 둔다.
5. 등록 끝나면 한 건 시험 생성해서 변수 치환·PDF 출력이 맞는지 확인한다.

## A. 신청내역(CSV)으로 일괄 생성 — 기본 흐름

"신청내역 읽어서 / CSV로 계약서 뽑아줘" 라고 하면:

1. `1.신청서넣기/`에서 가장 최근(또는 지정된) `.csv`를 읽는다.
   - CSV 헤더 = 변수명. 한 행 = 계약서 한 장. 여러 행이면 행마다 한 장씩 만든다.
   - 흔한 컬럼: `거래처, 교육과목, 교육일정, 건수, 건당금액, 계약일`.
     구글폼이 붙이는 타임스탬프 등 불필요 컬럼은 무시.
2. `2.계약서양식/`의 양식을 읽어 `{{ }}` 변수를 추출한다(양식 여러 개면 어떤 종류인지 확인).
3. 각 행에 대해:
   - 🔵는 `내정보.json`, 🟢는 행의 `거래처` 값으로 `거래처.json`에서 채움.
     거래처가 `거래처.json`에 없으면 사용자에게 묻고 "거래처.json에 저장할까요?" 제안.
   - 🟡는 CSV 행 값으로 채움. 빠진 칸만 사용자에게 되묻는다.
   - ⚙️ 총금액 = 건수 × 건당금액. 천단위 콤마, 한글금액(예: 120000 → "십이만") 변환.
     **건수·총금액이 본문 여러 곳에 나오면 전부 같은 값으로** 채운다(원본의 "3건 vs 4건"
     같은 불일치를 막는 게 이 도구의 핵심).
4. 모든 `{{변수}}`를 치환한 결과를 `4.완성계약서/`에 저장한다.
   미치환 `{{ }}`가 남으면 경고.
5. 파일명: `YYYY-MM-DD_거래처_종류_v버전.html`  예) `2026-06-16_넥스트러너스_특강계약_v1.html`
6. **HTML 저장 직후, 같은 이름으로 PDF를 자동 생성**한다(아래 "공통: PDF 자동 변환").
7. 끝나면 생성된 `.pdf` 목록을 안내 + "전자서명은 유캔싸인 등에 이 PDF를 업로드".

## B. 한 건만 즉석 생성

CSV 없이 "○○ 건 계약서 하나 뽑아줘" 라고 하면, 위 3~6을 한 행 분량으로만 수행.
🔵는 자동, 🟢는 거래처 골라, 🟡 건별만 하나씩 묻는다.

## C. 재계약 / 새 버전 (v2, v3…)

"○○ 재계약 / 지난 ○○ 계약 불러와" 라고 하면:

1. `4.완성계약서/`에서 그 거래처의 가장 최근 파일을 찾는다.
2. 그 값들을 기본값으로 깔고 **바뀐 조건만** 묻는다(보통 일정·건수·금액).
3. 같은 거래처의 최대 버전 +1로 저장: `..._v2.html`. 이전 버전은 지우지 않는다(이력).

## D. 이력 보기

"계약 이력 / 이번 달 계약 / ○○ 계약 목록" → `4.완성계약서/`를 훑어 표로:
날짜 · 거래처 · 종류 · 버전 · (가능하면)금액. 파일명만으로도 1차 요약 가능.

## 공통: PDF 자동 변환 (HTML을 만들면 항상 같이)

A·B·C 어느 흐름이든, HTML을 `4.완성계약서/`에 저장한 직후 **같은 이름의 PDF를 자동 생성**한다.
사용자가 ⌘P를 누를 필요 없다. macOS 헤드리스 크롬을 쓴다(검증됨: 한글·도장 자리·밑줄 정상, 3페이지 출력):

```bash
CHROME="/Applications/Google Chrome.app/Contents/MacOS/Google Chrome"
"$CHROME" --headless --disable-gpu --no-pdf-header-footer \
  --print-to-pdf="4.완성계약서/<같은이름>.pdf" \
  "file://$(pwd)/4.완성계약서/<같은이름>.html"
```

- Chrome이 다른 경로면 `which google-chrome` 등으로 찾고, 정 없으면 사용자에게 알린 뒤 ⌘P 수동 안내로 폴백.
- CSV 일괄이면 각 HTML마다 PDF를 만든다.

## 원칙

- **본문(약관) 절대 수정 금지.** 바꾸는 건 `{{변수}}`뿐.
- 🔵 내 고정값은 두 번 묻지 않는다. 🟡 건별만 묻는다.
- 한 번에 끝낼 수 있으면 한 번에. 빠진 변수만 콕 집어 되묻는다.
- 민감정보(주민번호·계좌)는 `3.내정보/`에만 두고 화면에 굳이 다시 출력하지 않는다.
- 새 양식(견적서 등)도 같은 방식 — `2.계약서양식/`에 HTML 하나 추가하면 끝.
